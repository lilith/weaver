# Weaver

A browser-based, AI-supported, collaborative world-building game engine. Successor to the original Weaver (lilith/weaver-lua, 2012).

Text-adventure-first (p/choose/goto, LoGD-style chat, free-text expansion) with per-location generated scene art. Every location, NPC, item, encounter, and UI theme is an entity — versioned, editable by prompt, and attributed to a pseudonym.

Designed for small family groups first, expanding to larger public worlds.

## Status

**Wave 0 — engine kernel spike.** Boostrapping the durable runtime, entity/component/relation store, and art generation loop. Not yet playable.

See [`spec/`](./spec/) for the full design — 12 documents covering architecture, location schema, inline scripts, expansion loop, world bible builder, testing trinity, wave plan, tech stack, theme generation, and prompt-based editing.

## Stack

- **Frontend** — SvelteKit 2 (Svelte 5 runes) + Vite 8 + Tailwind 4
- **Backend + DB** — Convex (managed)
- **LLM** — Claude Opus 4.7 (narrative, 1M context) / Sonnet 4.6 (dialogue) / Haiku 4.5 (intent classification, VLM eval)
- **Image gen** — FLUX.2 [pro] via fal.ai; FLUX Kontext for edits
- **Storage** — Cloudflare R2 (zero egress)
- **Hosting** — Cloudflare Pages
- **Auth** — Better Auth + Resend magic links

## Development

Prereqs: Node 22+, pnpm 9+.

```bash
# One-time
cp .env.example .env    # fill real secrets in .env
pnpm install

# Each session
npx convex dev          # in one terminal — pushes schema, watches backend
cd apps/play && pnpm dev  # in another — SvelteKit dev server
```

All secrets live in Convex dashboard env vars (`npx convex env set KEY value`). Client never sees API keys.
